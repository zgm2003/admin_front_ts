## 用户人工检查

- [ ] 离线本地 UI 加载：断开网络后启动安装版，确认应用壳与本地页面仍能加载，且未加载远程站点作为主 UI。
- [ ] 密码登录：恢复网络后使用密码正常登录、进入默认授权页面，并确认不会额外要求登录 Captcha。
- [ ] 浏览器与桌面凭证隔离：浏览器会话与桌面会话互不借用 refresh credential，桌面 DOM/存储中不可见 refresh credential。
- [ ] 重启与会话恢复：关闭并重新启动应用，确认桌面凭证可按正式 refresh 契约恢复会话；失效凭证明示为重新登录。
- [ ] Windows 凭证清理：退出登录后在 Windows Credential Manager 中确认 `cn.zgm2003.admin.refresh` / `current-session` 已删除。
- [ ] 受管下载允许/拒绝/取消：允许两个正式 HTTPS 主机；拒绝 HTTP、私网/回环、跨主机重定向、非法文件名；取消后无 `.part` 残留。
- [ ] 窗口关闭与最小化：最小化、最大化、关闭确认、记住选择、托盘恢复和用户确认退出均符合预期。
- [ ] 系统通知权限：允许与拒绝 Windows 通知权限各检查一次；通知正文为纯文本，托盘未读/闪烁可正确清除。
- [ ] 更新签名失败：使用无效签名测试清单时必须拒绝下载/安装，不显示带签名查询参数的原始错误。
- [ ] 无远程原生能力：远程页面、iframe 和外部支付页面均不能调用窗口、凭证、下载、通知或进程命令。
